# `catstat` — Known Issues & Limitations

Honest list of current limitations, intentional deferrals, and open risks. Severity: **S1** (blocks
correctness/leakage), **S2** (functional gap), **S3** (polish). Update as items are opened/closed.

## Status: 0.5.1 on PyPI (2026-07-02) — CPU + device-resident cuDF GPU
Most M0 deferrals are resolved; the remaining rows are honest environment notes. The leakage risk
(KI-011) is actively guarded by `tests/test_cross_fit_no_leakage.py` (OOF reconstruction is
exact) plus the `leakage-audit` gate; KI-010 (auto-smoothing parity) is resolved — catstat's
`smooth="auto"` is exactly sklearn's formula.

| KI-030 | — | ~~Numeric TE (0.2.0): `Count`/`Frequency` don't bin~~ | **Resolved 2026-06-27** (→ 0.4.0): `Count`/`Frequency` now bin numeric columns — `numeric`/`cardinality_threshold`/`n_bins`/`binning` added verbatim to both encoders, reusing `TargetEncoder`'s shared `_numeric.py` plumbing (**no `_base.py`/`_validation.py` edit** — `select_cols` already gates on `numeric_mode`, `_fit_count` already histograms string keys). | KI-031 | S3 | OOF slow path: order stats / custom re-fit per fold on CPU | Narrowed twice: **gather** replaced per-column `map` at transform (×2.3–3.4, 2026-06-27), and **skew/kurt moved onto the single-pass kernel** (order-4 power sums, 2026-07-02); on the **device** path median/min/max keep the per-fold loop but on resident data (only per-code tables leave the GPU, ×10–12.4 vs CPU at 1M–5M). ## Open risks to track (carry into implementation)
| id | sev | risk | mitigation |
|----|-----|------|-----------|
| KI-010 | — | ~~`smooth="auto"` exact formula unverified~~ | **Resolved 2026-07-02**: with sklearn 1.9 locally, catstat's EB (`m_i = σ²_pop_i/τ²_pop`, `λ = n/(n+m)`) matches sklearn `TargetEncoder(smooth='auto').encodings_` to fp rounding (max rel ≤ 4e-14; binary/constant-target bit-exact) across continuous/binary/multiclass, singletons, constant categories, 1e9 offsets, 500-cat high-card. Same formula (`λ = n·τ²/(n·τ² + SS/n)`). Guarded by `tests/test_sklearn_auto_parity.py` (skips < 1.4). |
| KI-011 | S1 | Leakage via implementation detail | OOF reconstruction test + `leakage-audit` skill gate every cross-fit/smoothing change. |
| KI-012 | S3 | sklearn `check_estimator` — documented subset | **2026-06-26**: applicable checks pass (`tests/test_check_estimator.py`, sklearn ≥ 1.6); inapplicable ones waived with reasons (sparse, 1d/empty/complex input, by-name `n_features`, y-messages). Estimator pickling fixed. |
| KI-013 | S3 | cuDF weak on object/high-cardinality strings | Largely mitigated: the device path factorizes strings to int64 codes at ingest and joins via device hash tables (validated to 250k categories on T4); `auto` keeps host-origin data on CPU anyway (KI-020). Residual: exotic object dtypes should go through `.to_pandas()`. |
| KI-014 | S3 | pandas↔cuDF NaN/dtype/numerics semantics differ | Parity is asserted at allclose by design. Concrete instance found + fixed 2026-07-02: cuDF's one-pass `var` vs pandas' two-pass cancelled *differently* at `\|mean\|≫sd` → all dispersion/shape/EB fit reductions are now shift-stable on both backends (offset parity test guards it). Null↔MISSING mapping validated (factorize `-1` remap). |
| KI-015 | S3 | Custom aggregations must be order-independent | warn otherwise; CPU-only; no smoothing. |
| KI-016 | — | ~~Multiclass column explosion for large `K`~~ | **Resolved 2026-07-02**: `max_classes` caps the one-vs-rest expansion to the most frequent classes (`encoded_classes_`/`target_mean_` aligned; ties break toward the earlier class); an uncapped `K > 100` target warns. Class-agnostic stats were already not `×K`. |
| KI-017 | S3 | Colab GPU sessions are fragile | Playbook baked into `scripts/colab_gpu_tests.py` (2026-07-02): prefer the image's **preinstalled RAPIDS** (blind `pip install cudf-cu12` can outrun the driver); **inherit the env** in subprocesses (stripping `LD_LIBRARY_PATH` → stub libcuda → `cudaErrorInsufficientDriver`); neutralize the dist-packages `tests` package shadowing; always run `colab exec` under an external watchdog (lost sessions hang it); session-assignment quota (`TooManyAssignmentsError`) cools off in ~hours — reuse one session. |
| KI-021 | — | ~~CI red: bare `pytest` can't import `tests`~~ | **Resolved 2026-06-26**: CI ran `pytest tests/` (not `python -m pytest`), so the repo root was off `sys.path` and `tests.conftest` failed to import. Added `pythonpath=["src","."]` to the pytest config. |
| KI-022 | — | ~~`cols="auto"` misses pandas ≥3.0 default string dtype~~ | **Resolved 2026-06-26**: `select_cols` now also selects pandas `StringDtype` (pandas 3.0 types strings as `StringDtype`, not `object`). Verified on sklearn 1.9 / pandas 3.0.3 — full suite green. |
